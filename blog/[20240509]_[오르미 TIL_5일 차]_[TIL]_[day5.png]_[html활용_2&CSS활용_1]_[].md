#### 테이블 태그
>필수 요소  

\<tr\>  
&nbsp;&nbsp;&nbsp;행을 구성하는 가장 기본 단위  
&nbsp;&nbsp;&nbsp;테이블은 tr이 쌓여서 형성
\<td\>  
&nbsp;&nbsp;&nbsp;테이블에서 각각의 데이터
\<th\>  
&nbsp;&nbsp;&nbsp;테이블 행, 열의 제목을 나타냄   
\<caption\>  
&nbsp;&nbsp;&nbsp;테이블의 제목 / 설명  
&nbsp;&nbsp;&nbsp;table의 첫번째 자식으로 사용해야 함
>부가 요소

\<thead\>  
&nbsp;&nbsp;&nbsp;thead안에 있는 요소들은 th 태그로 감싸져 있어야 함  
&nbsp;&nbsp;&nbsp;테이블의 제목 요소들은 다 th태그

\<colgroup\>  
&nbsp;&nbsp;&nbsp;열의 그룹  
  ```
  <colgroup>
        <col>
        <col span="2" class="batman">
        <col span="2" class="flash">
  </colgroup>
  ```
&nbsp;&nbsp;&nbsp;각각의 col은 table에서 첫번째 열, 두번째 열···을 의미하며, span-col이 선택하는 열이 어디까지인지 제어  
&nbsp;&nbsp;&nbsp;css 스타일을 공통적으로 주고싶을때 사용  
\<thead\>  
&nbsp;&nbsp;&nbsp;테이블의 머릿글(제목을 포함하는 요소)  
\<tbody\>  
&nbsp;&nbsp;&nbsp;테이블의 본문 (중요한 데이터들을 감쌈)  
\<tfoot\>  
&nbsp;&nbsp;&nbsp;테이블의 결론을 감쌈, 모든 테이블이 결론이 있는 건 아님  

#### 테이블 병합
- \<colspan\>  
  열 병합  
  열과 열을 더해 가로로 합쳐짐, **세로로 병합 X**  
  colspan="칸 개수"  
- \<rowspan\>  
  행 병합  
  행과 행을 더해 세로로 합쳐짐, **가로로 병합 X**  
  rowspan="칸 개수"  


## CSS
Cascading Style Sheets

#### 태그 선택자
- 선택자(selector)  
  어떤 html 태그에 적용될지 선택  
  속성 이름: 속성 값; 형태  ex) color: blue;  
  전체 태그가 아니라 특정 부분의 태그에만 style을 적용하고 싶으면 그 태그에 class(=요소의 이름)을 적용  
  클래스를 쓸 땐 앞에 .붙이기 -> .th_1 {

#### 여러 가지 CSS 적용법
1. 인라인
   태그 자체에 style 속성으로 스타일을 지정  
	ex) \<p style="color:yellow; . . .\>Hello World!\</p\>  
	:hover, ::before, ::after같은 가상요소에는 사용 불가  

2. 태그 선택자
   head 안에 \<style\> 적용  
   css파일과 html을 분리하는 것을 가장 권장  

3. 태그 선택자 with 그룹핑(그룹 선택자 ,)  
4. 상속  
	부모 요소에 css 적용  
   width, height, margin, padding, border은 상속되지 않음  
   input이나 button은 css를 상속받지 않고 브라우저마다 다르게 나타남  


#### 다양한 선택자
1. 전체 선택자
```css
*{
	margin:0;
	padding:0;
}
```
- 개발자가 모르는 브라우저만의 스타일을 제거하기 위해 사용  
2. 타입(=태그, 요소) 선택자
```css
h1{
	background-color: blue;
}
div{
	width: 20px;
}
```
3. id 선택자
```css
<header id="header>···</header>

#header{
	height: 30px;
}
```
- id, class 값은 숫자로 시작하면 안 됨 (네이밍 컨벤션)
- id는 html 페이지 내에 유일해야 함  
4. 클래스 선택자
```css
<h1 class="fc-red">hello wolrd</h1>
<p>Lorem ipsum dolor sit amt</p>
<p class="fc-red">Lorem ipsum dolor sit amt</p>

.fc-red {
	color: red;
}
```
5. 특성 선택자
```css
[type="button"]{
	border:0;
	cursor:pointer;
}
```
- 클래스도 가능  
6. 그룹 선택자
```css
h1, h2, h3, h4, h5, h6{ font-weight:bold;}
```


#### 복합 선택자
1. 자손(하위) 선택자
   공백으로 선택
   ```css
    div p button, h1{ }
	```
- div의 자식인 p의 자식인 button이랑 h1을 선택하겠다
- 모든 선택자는 조합 가능  
  

#### 가상 클래스
- 가상 클래스 선택자를 쓸 때는 콜론이 들어감
- :hover - 마우스 커서를 올려놓을 때  
- :link - 아직 들어가지 않은 태그의 스타일을 설정
- :visited - 이미 방문한 적 있는 링크의 스타일을 설정
- :active - 마우스를 클릭할 때
- :focus - 되는게 있고 안 되는게 있는데 보통 사용자랑 상호작용이 되는 애들은 포커싱이 됨  
  ex) a, button, input 등  
  

#### 선택자 우선순위
1. 후자 우선 법칙  
-뒤에 적힌게 먼저 적용
2. 구체성의 원칙  
-어떤 선택자가 더 구체적인가  
    1. inline 속성
    2. id #
    3. class ., 가상 클래스, 속성 선택자
    4. type(tag), 가상 요소 선택자
    5. 전체 선택자

&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;-점수 합산 값이 커도 우선순위는 변하지 않음  
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;-가중치를 계산해주는 사이트도 있음  
3. 중요성의 원칙  
-!important : 가중치 무시 -> 다른 어떤 CSS 보다도 우선  
-적용을 원하는 속성 뒤에 씀    
&nbsp;&nbsp;&nbsp;ex) color: green !important;  
-important는 뒤에 나오는 important만 덮을 수 있어서 사용을 지양해야 함  

#### display
- 블록/인라인 요소의 특징을 바꿀 수 있는 속성
- 다른 요소들과 어떻게 배치할지(요소와 요소사이의 관계)를 지정
- block: 블록 레벨 요소의 특징을 갖게 하겠다!
- inline: 인라인 요소의 특징을 갖게 하겠다!
- inline-block: 인라인-블록 레벨 요소의 특징을 갖게 하겠다!
- none: 아예 구조에서 빼버려서 보이지 않는 것
- visibility는 보이게, 안 보이게 설정하는 것
##### flex와 grid
- 자식 요소의 배치를 바꿀 때 부모 요소에 지정  
ex) div > ul > li 가 있을때 ul의 레이아웃을 바꾸고 싶을때 부모인 div의 display 속성을 flex나 grid로 바꿈  
- 요소와 그 내부 요소(자식)의 관계
- flex는 1차원적, grid는 2차원적으로 설정  

#### CSS Box Model
![box_model](img/content/boxmodel.png)
- box model: margin > border > padding > element  
##### width & height
- element의 크기 지정  
##### padding
- top-right-botton-left 순으로 지정  
&nbsp;&nbsp;4군데가 다 같으면 하나만 적음   
&nbsp;&nbsp;3개만 있으면 top - 좌우 - bottom  
&nbsp;&nbsp;2개만 있으면 위아래 - 좌우
##### border
- 요소의 테두리를 지정
- 테두리 모양, 굵기, 모서리 깎음 정도 등을 설정할 수 있음
##### margin
- 항상 투명, 색깔을 입힐 수 없음
- 다른 요소와의 간격을 의미
- padding처럼 네 방향으로 구분
- auto: 요소를 가운데 정렬 할 때 사용
##### box-sizing
- box의 사이즈를 측정하는 기준을 어디에 둘지 정하는 것
- content-box: 기본값으로 width, height만 해당하며 border, padding 포함하지 않음
- border-box: width, height에 border, padding까지 포함해서 계산 -> 주로 유니버셜 스타일에 추가됨
